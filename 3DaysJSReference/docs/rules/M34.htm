Variable declarations are not placed to the top of the scope

JavaScript does not have block scope. That means that variables declared within a block exist and are accessible in their containing environment. The following code is valid: 


function f()
{
	for (var i = 0; i < 10; i++)
	{
		alert(i);
	}
}

Declaring a variable in a block may be an indication of a potential misunderstanding and should be avoided. The following is better: 


function f()
{
	var i = 0;
	for (i = 0; i < 10; i++)
	{
		alert(i);
	}
}
