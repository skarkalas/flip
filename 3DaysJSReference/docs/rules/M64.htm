Wrong comparison with NaN

NaN is a special value of type Number that has a unique characteristic: it is not equal to anything, including itself. That implies that it is meaningless to be used in comparison operations. The following code is valid but meaningless (the condition evaluates always false): 


var x = parseInt("test", 10);
if (x === NaN)
{
    x = 10;
}

This problem can be fixed by using the function isNaN() instead. 


var x = parseInt("test", 10);
if (isNaN(x))
{
    x = 10;
}
